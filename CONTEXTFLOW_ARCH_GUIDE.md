# ContextFlow MVP Architecture Guide (Based on Repo B)

## 1. Project Vision
Transform linear ChatGPT conversations into a structured logic tree featuring "Categorization (Classifier)", "Summarization (Summarizer)", and "Branch Management (MergeUp)".

## 2. Core Architectural Modifications

### A. Data Model Extension (Source: Cynthia387)
Extend the existing node data structure in `src/shared/database/schema.js` or the Node definition to include:
- `category`: string (e.g., [Feature | Debug | Refactor | Other]) - Determined by the Classifier.
- `summary`: string[] - An array of summarized strings generated by the Summarizer, supporting multi-node merging.
- `status`: string (e.g., [active | archived]) - Used for logical branch management and "logical deletion" during MergeUp.

### B. Logic Layer Integration (Source: Cynthia387 logic files)
- **Classifier Service**: Triggered upon new message detection to assign a logic category to the node.
- **Summarizer Service**: Compresses the content of the current node into a concise summary.
- **MergeUp Logic**: Implements the function to push the current node's `summary` into its parent node and mark the current node as `archived`.

### C. UI & Visual Identity (Customization)
- **Theme Colors**: Replace Repo B's default blue with the primary color from the ContextFlow design specs.
- **Node Styling**: 
  - Implement **Category Tags** (colored labels) on top of each node.
  - Display a "Summary Preview" instead of the full message by default.
  - Maintain the **Dagre Layout** for automatic tree organization.

## 3. Critical File Mapping for Cursor Reference
